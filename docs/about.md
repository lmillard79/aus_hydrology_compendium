# About This Compendium

## What this is

The Australian Hydrology Compendium is a curated, annotated index of knowledge in Australian flood hydrology and water resources engineering. It is a navigable mind map of how topics and papers connect — a plain-language orientation layer for practitioners entering a topic area.

It is open and community-extendable.

## What this is not

- A repository of full papers (copyright issue — we link to sources, not reproduce them)
- A replacement for primary sources
- A static document — it is designed to grow over time

## Who it is for

Hydrologists, water resources engineers, flood practitioners, dam safety engineers, planners, and students — in Australia and internationally.

## Project rationale

The cumulative knowledge base of Australian hydrology is scattered across FMA conference proceedings, HWRS symposium papers, ANCOLD guidance, ARR chapters, BoM technical reports, and journal articles. Practitioners routinely reinvent solutions because they cannot find or navigate prior work. This compendium aims to make the knowledge graph visible, searchable, and linkable.

## Content sourcing workflow

1. Feed papers into NotebookLM by topic node
2. Use NotebookLM to identify main arguments, chronological development, and unresolved questions
3. Use NotebookLM output as a research scaffold only — do not publish directly
4. Write each page in plain language using the standard template
5. Verify access status for each reference (open / paywalled / contact author)
6. Commit to repository and deploy

## Key source organisations

| Organisation | Description | Notes |
|-------------|-------------|-------|
| FMA | Floodplain Management Australia conference proceedings | Partially open |
| HWRS | Hydrology and Water Resources Symposium proceedings | Partially open |
| ANCOLD | Australian National Committee on Large Dams | Guidance documents, some open |
| ARR | Australian Rainfall and Runoff (Engineers Australia) | Book chapters open at arr.ga.gov.au |
| BoM | Bureau of Meteorology | Data and technical reports, mostly open |
